Docs/Quickstart

Five-minute workflow

Register first, clean later.

Shelf works best when the artifact is registered the moment it is created. Start with explicit ledger and registry paths when testing.

quickstart
$ mkdir -p /tmp/shelf-demo
$ echo "debug output" > /tmp/shelf-demo/output.txt
$ shelf put /tmp/shelf-demo \
  --reason "quickstart artifact" \
  --ttl 1d \
  --kind scratch \
  --cleanup review \
  --ledger /tmp/shelf-ledger.jsonl \
  --registry /tmp/shelf-registry.json

1. Create Something Worth Tracking

mkdir -p /tmp/shelf-demo
echo "debug output" > /tmp/shelf-demo/output.txt

2. Put It On The Shelf

shelf put /tmp/shelf-demo \
  --reason "quickstart artifact" \
  --ttl 1d \
  --kind scratch \
  --cleanup review \
  --owner manual \
  --label quickstart \
  --ledger /tmp/shelf-ledger.jsonl \
  --registry /tmp/shelf-registry.json \
  --json

Capture the returned `id` anywhere future cleanup context matters.

3. Inspect The Ledger

shelf list --ledger /tmp/shelf-ledger.jsonl
shelf status --ledger /tmp/shelf-ledger.jsonl
shelf validate --ledger /tmp/shelf-ledger.jsonl --json
shelf due --ledger /tmp/shelf-ledger.jsonl --json

4. Preview Cleanup

shelf cleanup --dry-run --ledger /tmp/shelf-ledger.jsonl --json

Dry-run writes a plan under the ledger's `.shelf` directory only when cleanup entries exist. If there is nothing to clean up, it reports not-created and writes no plan file. Review any generated plan id before an execute step.

5. Stop Before Execute

shelf cleanup --execute --plan-id plan_20260601_120000_ab12
Execute is intentionally separate. Agents must not run it unless a human explicitly approves the reviewed plan id. After execution, Shelf writes a receipt and updates touched records so handled artifacts stop appearing in future due checks. Shelf also records generated plans and receipts in the ledger as Shelf-owned artifacts.
