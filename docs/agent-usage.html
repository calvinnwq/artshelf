04Agents · Overview

Create, monitor, review, clean.

An agent's whole relationship with Artshelf fits in four stages. Each stage has its own page with the exact commands; this page is the contract that ties them together.

Workflow summary

4.1 Create artshelf put <path> --reason <why> --ttl <ttl>Register a temporary file while you still remember why it exists, or state the reason for skipping it. 4.2 Monitor artshelf review --all --jsonCheck the shelf on a schedule with read-only commands, and stay quiet when everything is clear. 4.3 Review artshelf cleanup --dry-run --jsonPreview cleanup, then turn the output into an ArtshelfReviewReport decision packet a human can approve. 4.4 Clean artshelf cleanup --execute --plan-id <id>Run the one plan the human approved, clear old trash with its own reviewed purge plan, resolve confirmed ids, and verify the next review is quiet.

Trash rides the same loop. Monitor lists what sits in Artshelf trash, Review previews an age-based purge plan, and Clean executes only the reviewed purge plan id. Clearing trash never piggybacks on the cleanup plan that trashed the file.

Operating principles

agents remember
use the portable skill so final, status, and handoff turns check artifacts
crons only read
scheduled jobs may validate, review, dry-run, and report, but never execute
humans approve
mutation needs exact approval targets: ledger path, reviewed plan id, or record id list

The mental model

  • Green path Artifact is due, the dry-run plan is reviewed, and the approval target is exact.
  • Review first Artifact needs a human look, a retention change, or a keep/snooze decision.
  • Blocked Registry is stale, a path is suspicious, or the action would delete without a purge plan.

Portable skill

The repo ships a portable skill at skills/artshelf/SKILL.md. Agents that support local skills can copy or reference it directly.
