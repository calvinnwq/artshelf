Docs/Agent usage

For OpenClaw, Codex, Claude, and other agents

Register artifacts while intent is fresh.

Agents should use Shelf when temporary files need accountability, restart context, or later human review. Source files and cheap build outputs stay out.

agent contract
$ shelf put <path> \
  --reason "<why this exists>" \
  --ttl 3d \
  --kind run-artifact \
  --cleanup review \
  --owner openclaw \
  --label <project> \
  --json

Install Behavior

If shelf is not installed, ask where the user wants the repo cloned before setup. For now, use only the local clone, build, and npm link path. Do not choose an npm registry install or custom shim. 

git clone https://github.com/calvinnwq/shelf.git "$SHELF_REPO"
cd "$SHELF_REPO"
corepack enable
pnpm install --frozen-lockfile
pnpm run build
npm link
shelf --version

Use Shelf For

Backups

Rollback copies, pre-edit snapshots, migration backups, and quarantine folders.

Evidence

Debug output directories, generated reports, logs, and live-smoke artifacts.

Long runs

Workflow artifacts a future agent might need to resume, review, or clean up.

Skip Shelf For

  • Source files that belong in git.
  • Build outputs and dependency caches that can be regenerated cheaply.
  • Secrets, credential dumps, or private tokens.
  • Artifacts already owned by a more specific durable workflow ledger.

Report The ID

Shelf artifacts:
- shf_20260601_182800_ab12: /tmp/parser-output, debug evidence for SK-123,
  retain until 2026-06-04, cleanup=review

Put the id in handoffs, PR comments, Linear comments, daily memory, or task run summaries when the artifact matters for restart.

Cleanup Boundary

shelf validate --json
shelf due --json
shelf cleanup --dry-run --json

Those read-only review commands are fine for agents.

shelf cleanup --execute --plan-id <id>
Execution needs explicit human approval for the reviewed plan id. Never generate a fresh plan and execute it in the same step. Execution writes a receipt and updates touched ledger records to `trashed`, `review-required`, or `cleanup-refused`.

Agents may mark a ledger record manually resolved when the user confirms the artifact was inspected, is already missing, or is no longer needed. 

shelf resolve <id> --status resolved --reason <text>

Use a specific reason. resolve only updates the ledger; it does not move or delete files. Resolved records stop reappearing in future due and dry-run cleanup output while remaining visible through shelf list --status resolved.

Scheduled Review

Agents may schedule routine Shelf checks for stale artifacts through their host runtime, such as an agent cron, CI job, or recurring task. Keep scheduled jobs read-only. 

shelf validate --json
shelf due --json
shelf cleanup --dry-run --json

Reports should include the ledger path, due/manual-review/missing-path counts, cleanup dry-run plan id, executable entries, skipped entries, and refused entries. Stay quiet when nothing needs attention unless a regular summary was requested.

Use explicit ledger paths for scheduled checks. Do not scan arbitrary filesystem locations for ledgers unless the user opted into that discovery scope. Never schedule shelf cleanup --execute --plan-id <id>.
