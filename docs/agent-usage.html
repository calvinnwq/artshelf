Docs/Agent usage

For local agents and automation

Register artifacts while intent is fresh.

Agents should use Artshelf when temporary files need accountability, restart context, or later human review. Source files and cheap build outputs stay out.

agent contract
$ artshelf put <path> \
  --reason "<why this exists>" \
  --ttl 3d \
  --kind run-artifact \
  --cleanup review \
  --owner agent \
  --label <project> \
  --json

Registration Trigger

Treat Artshelf as a finalization check, not an optional cleanup habit. Before an agent reports a task as done, it should check whether the task created, copied, exported, quarantined, backed up, or preserved any non-source file or directory that may outlive the current command.

Register eligible artifacts immediately. If an eligible artifact is skipped, state the reason: source-controlled, regeneratable, secret-bearing, already tracked by another durable ledger, or explicitly not retained by request.

Install Behavior

If artshelf is not installed, prefer the package-manager install when available and verify the CLI before registering artifacts. 

npm install -g artshelf
artshelf --version
artshelf doctor
pnpm add -g artshelf
artshelf --version
artshelf doctor

For source installs, ask where the user wants the repo cloned before setup. Use the local clone, build, and npm link path. Do not create a custom shim. 

git clone https://github.com/calvinnwq/artshelf.git "$ARTSHELF_REPO"
cd "$ARTSHELF_REPO"
corepack enable
pnpm install --frozen-lockfile
pnpm run build
npm link
artshelf --version
artshelf doctor

Use Artshelf For

Backups

Rollback copies, pre-edit snapshots, migration backups, and quarantine folders.

Evidence

Debug output directories, generated reports, logs, and live-smoke artifacts.

Long runs

Workflow artifacts a future agent might need to resume, review, or clean up.

Skip Artshelf For

  • Source files that belong in git.
  • Build outputs and dependency caches that can be regenerated cheaply.
  • Secrets, credential dumps, or private tokens.
  • Artifacts already owned by a more specific durable workflow ledger.

Idempotent Lookup

Integrations should query the ledger before creating another record for the same artifact. find and get are read-only; they never move, resolve, or delete files. 

artshelf find --path <path> --owner <agent-or-runtime> --label <task-or-run-id> --json
artshelf get <id> --json

find requires at least one selector. Multiple labels are an all-label match. If it returns a record, reuse that Artshelf id; otherwise call artshelf put and record the new id.

Ledger Registry

Artshelf keeps a user-level registry at `~/.artshelf/ledgers.json` so one CLI can review all known ledgers without moving project records into one global file. put registers the ledger it writes to. 

artshelf ledgers add --ledger <repo>/.artshelf/ledger.jsonl --name <project> --scope repo
artshelf ledgers list --json
artshelf review --all --json
artshelf status --all --json
artshelf find --all --owner <agent-or-runtime> --json
artshelf trash list --all --json

Renamed installs before 0.5.0 used .shelf paths. For migration, copy the old ledger directories into .artshelf, rewrite registry entries, validate the new registry, and keep the .shelf copies until rollback is no longer needed.

artshelf ledgers list --json validates each registered ledger and reports ok/missing/invalid status with entry and warning/error counts, so agents can detect stale registry entries without a separate validate pass; add --plain for a fast listing that skips validation. artshelf review --all --json returns an aggregate triage summary and the next safe action next to the per-ledger detail.

Use global cleanup dry-run when you want Artshelf to write cleanup plans for registered ledgers with cleanup entries, without moving files.

artshelf cleanup --dry-run --all --json
--all is for discovery and review. Cleanup execution remains ledger-specific and requires a reviewed plan id for that ledger.

If the executable cleanup entries have not changed, dry-run reuses the existing plan id and refreshes the same plan file instead of creating duplicate plans.

Daily Review Workflow

Use this flow when a scheduled review, recurring task, or user request reports Artshelf cleanup attention.

  1. Register artifacts early during work, or state why an eligible artifact was skipped.
  2. Review state with read-only commands first: artshelf ledgers list --json, artshelf review --all --json, and artshelf trash list --all --json; for old trash on a selected ledger, run artshelf trash purge --older-than 7d --dry-run --ledger <ledger-path> --json.
  3. Present a decision packet instead of raw counts. Include registry health, affected ledgers, due/manual-review/missing-path counts, executable entries, skipped entries, refused entries, trashed record counts and ages, purge dry-run plan ids/skipped entries, and the next safe action.
  4. Classify each candidate as trash-safe, needs-human-review, resolve-candidate, or registry-problem.
  5. If cleanup execution is appropriate, generate or reuse a dry-run plan, then ask for explicit approval naming the ledger path and reviewed plan id.
  6. For trashed records, require a separate reviewed purge plan before physical deletion.
  7. After approved cleanup execute, trash purge, or resolve, verify quiet with artshelf review --all --json, plus artshelf trash list --ledger <ledger-path> --json and purge receipt evidence after purge, or explain what remains.
artshelf trash list --ledger <ledger-path>
artshelf trash purge --older-than 7d --dry-run --ledger <ledger-path> --json
artshelf trash purge --execute --plan-id <purge-plan-id> --ledger <ledger-path> --json
approve artshelf cleanup ledger <ledger-path> plan <plan-id>
approve artshelf trash purge ledger <ledger-path> plan <purge-plan-id>
Never execute from a read-only preview id. Never generate a fresh plan and execute it in the same step. trash moves artifacts into Artshelf trash; physical delete requires a separate reviewed trash purge plan.

Report The ID

Artshelf artifacts:
- shf_20260601_182800_ab12: /tmp/parser-output, debug evidence for issue-123,
  retain until 2026-06-04, cleanup=review

Put the id in handoffs, PR comments, issue comments, memory, or task run summaries when the artifact matters for restart.

If there are no eligible artifacts, say nothing. If eligible artifacts were skipped instead of registered, include the brief skip reason from the completion checklist.

Cleanup Boundary

artshelf validate --json
artshelf validate --all --json
artshelf due --json
artshelf due --all --json
artshelf review --all --json

Cleanup dry-run is safe to run. It writes plan files for later review only when there are executable cleanup entries; no-op dry-runs report not-created and write no plan file. Matching dry-runs reuse the existing plan id and refresh the plan timestamp.

artshelf cleanup --dry-run --json
artshelf cleanup --dry-run --all --json
artshelf cleanup --execute --plan-id <id>
artshelf trash list --ledger <ledger-path> --json
artshelf trash purge --older-than 7d --dry-run --ledger <ledger-path> --json
artshelf trash purge --execute --plan-id <purge-plan-id> --ledger <ledger-path> --json
Execution is approval-only: no daemon, no auto-execute, no global execute, and no fresh-plan-then-execute shortcut. Cleanup execution needs explicit human approval for the reviewed plan id. Trash list and purge dry-run are review steps; trash purge execution needs separate approval naming the ledger and reviewed purge plan id. Execution writes a receipt and updates touched ledger records to `trashed`, `review-required`, or `cleanup-refused`. cleanup=delete stays refused instead of silently deleting files; physical deletion requires a separate reviewed trash purge plan. Artshelf records generated plans and receipts as `owner=artshelf` artifacts.

Agents may mark a ledger record manually resolved when the user confirms the artifact was inspected, is already missing, or is no longer needed. 

artshelf resolve <id> --status resolved --reason <text>

Use a specific reason. resolve only updates the ledger; it does not move or delete files. Resolved records stop reappearing in future due and dry-run cleanup output while remaining visible through artshelf list --status resolved.

Scheduled Review

Agents may schedule routine Artshelf checks for stale artifacts through their host runtime, such as an agent cron, CI job, or recurring task. Keep scheduled jobs non-destructive. 

artshelf validate --json
artshelf validate --all --json
artshelf due --json
artshelf due --all --json
artshelf review --all --json

Read-only health and dashboard checks are also safe to schedule. Run artshelf review --all --json for aggregate triage (summary and nextAction), artshelf doctor --json to catch a broken or stale registry before relying on cleanup planning, and artshelf status --all --json for a compact cron summary. 

artshelf doctor --json
artshelf status --all --json

Scheduled cleanup and trash purge dry-runs may write plan files for later review when entries exist, but must not move or delete files. Matching cleanup dry-runs reuse the existing plan id and refresh the plan timestamp. 

artshelf cleanup --dry-run --json
artshelf cleanup --dry-run --all --json
artshelf trash list --ledger <ledger-path> --json
artshelf trash list --all --json
artshelf trash purge --older-than 7d --dry-run --ledger <ledger-path> --json

Reports should include the ledger path, due/manual-review/missing-path counts, cleanup dry-run plan id, executable entries, skipped entries, and refused entries. Trash reports may use artshelf trash list --all --json to discover trashed records across registered ledgers, then include trashed record counts and target ages. Run purge dry-runs only for an explicit ledger and report any plan id, matching entries, and skipped entries. Stay quiet when nothing needs attention unless a regular summary was requested.

Use explicit ledger paths for scheduled checks. Do not scan arbitrary filesystem locations for ledgers unless the user opted into that discovery scope. Never schedule cleanup execution or trash purge execution; scheduled jobs may only dry-run and report plans for later human review.

Completion Checklist

  1. Did you create, copy, export, quarantine, back up, or preserve any non-source file or directory?
  2. Will any of those paths outlive this command?
  3. If yes, did you register them with Artshelf or state why Artshelf is not appropriate?
Do not call work done while known eligible artifacts are neither registered nor explicitly skipped.
