Docs/Agent usage

For local agents and automation

Register artifacts while intent is fresh.

Agents should use Shelf when temporary files need accountability, restart context, or later human review. Source files and cheap build outputs stay out.

agent contract
$ shelf put <path> \
  --reason "<why this exists>" \
  --ttl 3d \
  --kind run-artifact \
  --cleanup review \
  --owner agent \
  --label <project> \
  --json

Registration Trigger

Treat Shelf as a finalization check, not an optional cleanup habit. Before an agent reports a task as done, it should check whether the task created, copied, exported, quarantined, backed up, or preserved any non-source file or directory that may outlive the current command.

Register eligible artifacts immediately. If an eligible artifact is skipped, state the reason: source-controlled, regeneratable, secret-bearing, already tracked by another durable ledger, or explicitly not retained by request.

Install Behavior

If shelf is not installed, ask where the user wants the repo cloned before setup. For now, use only the local clone, build, and npm link path. Do not choose an npm registry install or custom shim. 

git clone https://github.com/calvinnwq/shelf.git "$SHELF_REPO"
cd "$SHELF_REPO"
corepack enable
pnpm install --frozen-lockfile
pnpm run build
npm link
shelf --version

Use Shelf For

Backups

Rollback copies, pre-edit snapshots, migration backups, and quarantine folders.

Evidence

Debug output directories, generated reports, logs, and live-smoke artifacts.

Long runs

Workflow artifacts a future agent might need to resume, review, or clean up.

Skip Shelf For

  • Source files that belong in git.
  • Build outputs and dependency caches that can be regenerated cheaply.
  • Secrets, credential dumps, or private tokens.
  • Artifacts already owned by a more specific durable workflow ledger.

Idempotent Lookup

Integrations should query the ledger before creating another record for the same artifact. find and get are read-only; they never move, resolve, or delete files. 

shelf find --path <path> --owner <agent-or-runtime> --label <task-or-run-id> --json
shelf get <id> --json

find requires at least one selector. Multiple labels are an all-label match. If it returns a record, reuse that Shelf id; otherwise call shelf put and record the new id.

Ledger Registry

Shelf keeps a user-level registry at `~/.shelf/ledgers.json` so one CLI can review all known ledgers without moving project records into one global file. put registers the ledger it writes to. 

shelf ledgers add --ledger <repo>/.shelf/ledger.jsonl --name <project> --scope repo
shelf ledgers list --json
shelf review --all --json
shelf find --all --owner <agent-or-runtime> --json

Use global cleanup dry-run when you want Shelf to write cleanup plans for registered ledgers with cleanup entries, without moving files.

shelf cleanup --dry-run --all --json
--all is for discovery and review. Cleanup execution remains ledger-specific and requires a reviewed plan id for that ledger.

If the executable cleanup entries have not changed, dry-run reuses the existing plan id and refreshes the same plan file instead of creating duplicate plans.

Report The ID

Shelf artifacts:
- shf_20260601_182800_ab12: /tmp/parser-output, debug evidence for issue-123,
  retain until 2026-06-04, cleanup=review

Put the id in handoffs, PR comments, issue comments, memory, or task run summaries when the artifact matters for restart.

If there are no eligible artifacts, say nothing. If eligible artifacts were skipped instead of registered, include the brief skip reason from the completion checklist.

Cleanup Boundary

shelf validate --json
shelf validate --all --json
shelf due --json
shelf due --all --json

Cleanup dry-run is safe to run. It writes plan files for later review only when there are executable cleanup entries; no-op dry-runs report not-created and write no plan file. Matching dry-runs reuse the existing plan id and refresh the plan timestamp.

shelf cleanup --dry-run --json
shelf cleanup --dry-run --all --json
shelf cleanup --execute --plan-id <id>
Execution needs explicit human approval for the reviewed plan id. Never generate a fresh plan and execute it in the same step. Execution writes a receipt and updates touched ledger records to `trashed`, `review-required`, or `cleanup-refused`. Shelf records generated plans and receipts as `owner=shelf` artifacts.

Agents may mark a ledger record manually resolved when the user confirms the artifact was inspected, is already missing, or is no longer needed. 

shelf resolve <id> --status resolved --reason <text>

Use a specific reason. resolve only updates the ledger; it does not move or delete files. Resolved records stop reappearing in future due and dry-run cleanup output while remaining visible through shelf list --status resolved.

Scheduled Review

Agents may schedule routine Shelf checks for stale artifacts through their host runtime, such as an agent cron, CI job, or recurring task. Keep scheduled jobs non-destructive. 

shelf validate --json
shelf validate --all --json
shelf due --json
shelf due --all --json

Read-only health and dashboard checks are also safe to schedule. Run shelf doctor --json to catch a broken or stale registry before relying on cleanup planning, and shelf status --all --json for a compact cron summary. 

shelf doctor --json
shelf status --all --json

Scheduled cleanup dry-run may write plan files for later review when cleanup entries exist, but must not move or delete files. Matching dry-runs reuse the existing plan id and refresh the plan timestamp. 

shelf cleanup --dry-run --json
shelf cleanup --dry-run --all --json

Reports should include the ledger path, due/manual-review/missing-path counts, cleanup dry-run plan id, executable entries, skipped entries, and refused entries. Stay quiet when nothing needs attention unless a regular summary was requested.

Use explicit ledger paths for scheduled checks. Do not scan arbitrary filesystem locations for ledgers unless the user opted into that discovery scope. Never schedule shelf cleanup --execute --plan-id <id>.

Completion Checklist

  1. Did you create, copy, export, quarantine, back up, or preserve any non-source file or directory?
  2. Will any of those paths outlive this command?
  3. If yes, did you register them with Shelf or state why Shelf is not appropriate?
Do not call work done while known eligible artifacts are neither registered nor explicitly skipped.
