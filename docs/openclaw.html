OpenClaw local install

Use Shelf from a source checkout.

Shelf 0.1.0 is released for GitHub/source-first dogfooding. This mirrors Calvin's OpenClaw setup and keeps npm publishing out of the path.

openclaw
$ cd /Users/ngxcalvin/repos/shelf
$ pnpm install --frozen-lockfile
$ pnpm run build
$ node dist/src/cli.js --version

Copy-Paste Install

set -euo pipefail

SHELF_REPO="${SHELF_REPO:-$HOME/repos/shelf}"

if [ ! -d "$SHELF_REPO/.git" ]; then
  mkdir -p "$(dirname "$SHELF_REPO")"
  git clone https://github.com/calvinnwq/shelf.git "$SHELF_REPO"
fi

cd "$SHELF_REPO"
git pull --ff-only
corepack enable
pnpm install --frozen-lockfile
pnpm run build
node dist/src/cli.js --version

Optional Shim

mkdir -p "$HOME/.local/bin"
cat > "$HOME/.local/bin/shelf" <<'EOF'
#!/usr/bin/env bash
exec node "$HOME/repos/shelf/dist/src/cli.js" "$@"
EOF
chmod +x "$HOME/.local/bin/shelf"

export PATH="$HOME/.local/bin:$PATH"
shelf --version

Agents can still use the explicit `node .../dist/src/cli.js` form when PATH is uncertain.

Smoke Registration

SHELF_REPO="${SHELF_REPO:-$HOME/repos/shelf}"
SHELF_CMD="${SHELF_CMD:-node $SHELF_REPO/dist/src/cli.js}"
ARTIFACT="$(mktemp -d /tmp/shelf-openclaw-smoke.XXXXXX)"

$SHELF_CMD put "$ARTIFACT" \
  --reason "OpenClaw local Shelf setup smoke" \
  --ttl 3d \
  --kind run-artifact \
  --cleanup review \
  --owner openclaw \
  --label shelf \
  --label openclaw \
  --json

$SHELF_CMD validate --json
$SHELF_CMD due --json
$SHELF_CMD cleanup --dry-run --json
Do not run `cleanup --execute` without explicit human approval for the reviewed plan id.
