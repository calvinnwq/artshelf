Temporary artifacts · Accountable cleanup

Put run outputs somewhere accountable.

Shelf records why temporary artifacts, backups, and debug evidence exist, then gives humans and agents a reviewable cleanup plan before anything moves.

Why Shelf Exists

Agent-heavy projects create a lot of temporary stuff: copied files, rollback folders, smoke-test output, reports, and logs. Filesystem age cannot tell a future agent whether those artifacts are still useful. Shelf captures intent at creation time, while the reason is still fresh.

What Shelf Does

Register artifacts

Record path, reason, owner, labels, retention, and cleanup mode with `shelf put`.

Review due items

Use `shelf due` and `shelf cleanup --dry-run` to inspect what needs attention.

Execute by plan

Cleanup execution uses a reviewed plan id. V1 trashes safe entries and refuses delete.

Safety Model

  • Ledger-first. Shelf only cleans entries that were registered intentionally.
  • Dry-run before mutation. Plans are written before execution.
  • Execute by reviewed plan id. Agents should stop before `cleanup --execute`.
  • V1 refuses physical delete. `cleanup=review` stays review-only.
