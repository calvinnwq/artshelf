V1 command surface

Small commands, explicit cleanup.

Shelf v1 keeps the API narrow: put entries in a ledger, query existing records, write a cleanup plan, and execute only by reviewed plan id.

reference
$ shelf help
$ shelf help put
$ shelf help cleanup
$ shelf --version

Commands

shelf put <path>

Append a ledger entry with reason, retention, owner, labels, kind, and cleanup mode.

shelf list

Show ledger entries and current status. Use --status active|review-required|trashed|cleanup-refused|resolved to filter.

shelf find --path <path>

Read-only lookup for existing records by path, owner, labels, and status.

shelf get <id>

Read-only audit lookup for one ledger record by Shelf id.

shelf due

Classify active entries as kept, due, manual-review, or missing-path.

shelf validate

Validate JSONL shape and warn about missing active or review-required paths.

shelf cleanup --dry-run

Create a cleanup plan without moving files.

shelf cleanup --execute --plan-id <id>

Execute a previously reviewed plan id, write a receipt, and update touched ledger records.

shelf resolve <id> --status resolved --reason <text>

Mark a handled, missing, or no-longer-needed record as manually resolved.

Global Options

--ledger <path>   Use an explicit JSONL ledger
--json            Emit machine-readable JSON
--help            Show help
--version         Show version

Retention

Choose exactly one retention mode when calling `put`.

--ttl 3d
--retain-until 2026-06-04T08:28:00Z
--manual-review

Cleanup Modes

review

Default. Included for review but not moved by execute.

trash

Moved into Shelf's local trash folder when the approved plan executes.

delete

Accepted in the ledger but refused by v1 execution.

Executed records move out of the active cleanup flow. `review` becomes `review-required`, `trash` becomes `trashed`, and refused delete becomes `cleanup-refused`. Manual resolution becomes `resolved`. `shelf list` keeps those records visible for audit.

Lookup

Use find before put when an integration needs idempotent artifact registration. It requires at least one selector and never mutates records. 

shelf find --path <path> --owner coding-workflow-pipeline --label <run-id> --json
shelf get <id> --json

Storage

Inside a git repo, Shelf defaults to `.shelf/ledger.jsonl`. Outside a repo, it defaults to `~/.shelf/ledger.jsonl`. Pass `--ledger <path>` for tests, demos, and controlled smoke runs.
