V1 command surface

Small commands, explicit cleanup.

Shelf v1 keeps the API narrow: put entries in a ledger, query existing records, write a cleanup plan, and execute only by reviewed plan id.

reference
$ shelf help
$ shelf help put
$ shelf help cleanup
$ shelf --version

Commands

shelf put <path>

Append a ledger entry with reason, retention, owner, labels, kind, and cleanup mode.

shelf ledgers list

Show the user-level registry of known repo and user ledgers.

shelf ledgers add --ledger <path> [--scope repo|user|other]

Register an existing ledger so Shelf can review it through one entry point. Scope is inferred from the path when omitted.

shelf list

Show ledger entries and current status. Use --all and --status active|review-required|trashed|cleanup-refused|resolved to filter registered ledgers.

shelf find --path <path>

Read-only lookup for existing records by path, owner, labels, and status.

shelf get <id>

Read-only audit lookup for one ledger record by Shelf id. Use --all to search registered ledgers.

shelf due

Classify active entries as kept, due, manual-review, or missing-path. Use --all to read every registered ledger after registry validation.

shelf validate

Validate JSONL shape and warn about missing active or review-required paths. Use --all to report stale registered ledgers.

shelf review

Run validate, due, and cleanup plan preview. Invalid ledgers and valid ledgers with no cleanup entries report a not-created plan.

shelf doctor

Report machine and registry health: CLI version, default ledger and registry paths, stale or invalid registered ledgers, and the cleanup safety posture. Read-only; run it after install or when --all commands misbehave. Exits non-zero when the registry or a registered ledger is broken.

shelf status

Lightweight daily dashboard: registry health, total ledgers, and counts of active, kept, due, manual-review, and missing-path entries plus the pending cleanup count. Use --all for every registered ledger. Read-only and never writes plans or receipts; --all --json is cron-friendly and human output is short enough to paste into a chat.

shelf cleanup --dry-run

Create and register a cleanup plan without moving files when cleanup entries exist; no-op dry-runs report not-created, and matching dry-runs reuse the existing plan id.

shelf cleanup --execute --plan-id <id>

Execute a previously reviewed plan id, write a receipt, register the receipt artifact, and update touched ledger records.

shelf resolve <id> --status resolved --reason <text>

Mark a handled, missing, or no-longer-needed record as manually resolved.

Global Options

--ledger <path>   Use an explicit JSONL ledger
--registry <path> Use an explicit ledger registry
--all             Read all registered ledgers for supported commands
SHELF_REGISTRY    Override the default ledger registry path
--json            Emit machine-readable JSON
--help            Show help
--version         Show version

Retention

Choose exactly one retention mode when calling `put`.

--ttl 3d
--retain-until 2026-06-04T08:28:00Z
--manual-review

Cleanup Modes

review

Default. Included for review but not moved by execute.

trash

Moved into Shelf's local trash folder when the approved plan executes.

delete

Accepted in the ledger but refused by v1 execution.

Executed records move out of the active cleanup flow. `review` becomes `review-required`, `trash` becomes `trashed`, and refused delete becomes `cleanup-refused`. Manual resolution becomes `resolved`. `shelf list` keeps those records visible for audit.

Lookup

Use find before put when an integration needs idempotent artifact registration. It requires at least one selector and never mutates records. 

shelf find --path <path> --owner <agent-or-runtime> --label <task-or-run-id> --json
shelf find --all --owner <agent-or-runtime> --json
shelf get <id> --json
shelf get <id> --all --json

Storage

Inside a git repo, Shelf defaults to `.shelf/ledger.jsonl`. Outside a repo, it defaults to `~/.shelf/ledger.jsonl`. Pass `--ledger <path>` for tests, demos, and controlled smoke runs.

Shelf also keeps a user-level registry at `~/.shelf/ledgers.json`. Override it with --registry <path> or SHELF_REGISTRY. The registry is a discovery index for `--all` review commands; project records stay in their own repo-local ledgers. 

shelf ledgers add --ledger <repo>/.shelf/ledger.jsonl --name <project> --scope repo
shelf review --all --json
shelf cleanup --dry-run --all --json

shelf ledgers add requires an existing ledger path. Scope may be repo, user, or other; when omitted, Shelf infers it from the path. All-mode reads report stale or invalid registered ledgers before returning records. Global cleanup dry-run writes plan files only for ledgers with cleanup entries, while global cleanup execution is refused. Execute only a specific reviewed plan id against the ledger that produced it.
