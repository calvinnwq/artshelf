V1 command surface

Small commands, explicit cleanup.

Artshelf v1 keeps the API narrow: put entries in a ledger, query existing records, write a cleanup plan, and use separate reviewed plan ids for cleanup execution and trash purge.

reference
$ artshelf help
$ artshelf help put
$ artshelf help cleanup
$ artshelf --version

Commands

artshelf put <path>

Append a ledger entry with reason, retention, owner, labels, kind, and cleanup mode.

artshelf ledgers list

List registered ledgers with per-ledger validation status (ok/missing/invalid), entry counts, and warning/error counts so agents can spot stale registry entries without a separate validate pass. Exits non-zero when the registry or any ledger is broken. Add --plain for the fast listing that skips validation.

artshelf ledgers add --ledger <path> [--scope repo|user|other]

Register an existing ledger so Artshelf can review it through one entry point. Scope is inferred from the path when omitted.

artshelf list

Show ledger entries and current status. Use --all and --status active|review-required|trashed|cleanup-refused|resolved to filter registered ledgers.

artshelf find --path <path>

Read-only lookup for existing records by path, owner, labels, and status.

artshelf get <id>

Read-only audit lookup for one ledger record by Artshelf id. Use --all to search registered ledgers.

artshelf due

Classify active entries as kept, due, manual-review, or missing-path. Use --all to read every registered ledger after registry validation.

artshelf validate

Validate JSONL shape, handled cleanup metadata, and warn about missing active, review-required, or trashed target paths. Use --all to report stale registered ledgers.

artshelf review

Run validate, due, and cleanup plan preview. Invalid ledgers and valid ledgers with no cleanup entries report a not-created plan. --all adds an aggregate triage summary (affected ledgers, due, manual-review, missing-path, executable, and skipped counts plus preview plan ids) and states the next safe action, staying read-only.

artshelf doctor

Report machine and registry health: CLI version, selected/default ledger path, selected/global registry path, stale or invalid registered ledgers, and the cleanup safety posture. Read-only; run it after install or when --all commands misbehave. Exits non-zero when the registry or a registered ledger is broken.

artshelf status

Lightweight daily dashboard: single-ledger counts by default, or registry health, total ledgers, and aggregated counts with --all. Reports active, kept, due, manual-review, and missing-path entries plus the pending cleanup count. Read-only and never writes plans or receipts; --all --json is cron-friendly and human output is short enough to paste into a chat.

artshelf cleanup --dry-run

Create and register a cleanup plan without moving files when cleanup entries exist; no-op dry-runs report not-created, and matching dry-runs reuse the existing plan id.

artshelf cleanup --execute --plan-id <id>

Approval-only execution for one reviewed plan id: no daemon, no auto-execute, no global execute, and no fresh live set during execute. Writes a receipt, registers the receipt artifact, and updates touched ledger records.

artshelf trash list [--all] [--ledger <path>] [--json]

List trashed records with target path, receiving receipt provenance, and age.

artshelf trash purge --older-than <ttl> --dry-run [--ledger <path>] [--json]

Create a reviewed age-based trash purge plan from a specific ledger. The reviewed plan can be executed with --execute --plan-id <id>.

artshelf trash purge --execute --plan-id <id> [--ledger <path>] [--json]

Execute a reviewed trash purge plan id for one explicit ledger. --all is not supported for purge; completed receipts are refused, while started receipts may be resumed and reconciled.

artshelf resolve <id> --status resolved --reason <text>

Mark a handled, missing, or no-longer-needed record as manually resolved.

Global Options

--ledger <path>   Use an explicit JSONL ledger
--registry <path> Use an explicit ledger registry
--all             Read all registered ledgers for supported commands
ARTSHELF_REGISTRY    Override the default ledger registry path
SHELF_REGISTRY       Legacy fallback when ARTSHELF_REGISTRY is unset
--json            Emit machine-readable JSON
--help            Show help
--version         Show version

Retention

Choose exactly one retention mode when calling `put`.

--ttl 3d
--retain-until 2026-06-04T08:28:00Z
--manual-review

Cleanup Modes

review

Default. Included for review but not moved by execute.

trash

Moved into Artshelf's local trash folder when the approved plan executes.

delete

Accepted in the ledger but refused by v1 execution.

Executed records move out of the active cleanup flow. `review` becomes `review-required`, `trash` becomes `trashed`, and refused delete becomes `cleanup-refused`. Manual resolution becomes `resolved`. `artshelf list` keeps those records visible for audit.

`trash` moves stay in Artshelf quarantine until you run `artshelf trash purge` with a reviewed purge plan for the current ledger.

Lookup

Use find before put when an integration needs idempotent artifact registration. It requires at least one selector and never mutates records. 

artshelf find --path <path> --owner <agent-or-runtime> --label <task-or-run-id> --json
artshelf find --all --owner <agent-or-runtime> --json
artshelf get <id> --json
artshelf get <id> --all --json

Storage

Inside a git repo, Artshelf defaults to `.shelf/ledger.jsonl`. Outside a repo, it defaults to `~/.shelf/ledger.jsonl`. Pass `--ledger <path>` for tests, demos, and controlled smoke runs.

Artshelf also keeps a user-level registry at `~/.shelf/ledgers.json`. Override it with --registry <path> or ARTSHELF_REGISTRY; legacy SHELF_REGISTRY is used only when ARTSHELF_REGISTRY is unset. The registry is a discovery index for supported `--all` review, status, cleanup dry-run, and trash-list commands; project records stay in their own repo-local ledgers. 

artshelf ledgers add --ledger <repo>/.shelf/ledger.jsonl --name <project> --scope repo
artshelf review --all --json
artshelf status --all --json
artshelf cleanup --dry-run --all --json
artshelf trash list --all --json

artshelf ledgers add requires an existing ledger path. Scope may be repo, user, or other; when omitted, Artshelf infers it from the path. All-mode reads report stale or invalid registered ledgers before returning records. Global cleanup dry-run writes plan files only for ledgers with cleanup entries, and artshelf trash list --all --json is read-only trash discovery. Global cleanup execution and trash purge with --all are refused. Execute only a specific reviewed plan id against the ledger that produced it.
